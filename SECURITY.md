# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly
by emailing **<venkrbec.tomas@gmail.com>** with the subject line
`[lazyline] Security vulnerability`.

Please **do not** open a public GitHub issue for security vulnerabilities.

You can expect:

- An acknowledgment within **48 hours**.
- A follow-up with an assessment and timeline within **7 days**.
- Credit in the fix advisory (unless you prefer to remain anonymous).

## Security Considerations

Lazyline is a **development tool** intended to run locally in trusted
environments. It is not designed for production use or execution on
untrusted input. The following aspects are relevant:

### Pickle deserialization

Worker processes (multiprocessing and subprocess profiling) exchange
profiling stats via pickle files in temporary directories. The
directories are created with restrictive permissions (`0700`) and files
with `0600`. Each directory name includes a random suffix generated by
`tempfile.mkdtemp`, making blind enumeration impractical. However,
pickle deserialization is inherently unsafe against malicious input —
a same-user process that discovers the temp directory path could inject
crafted pickle files. This is an upstream constraint of
`line_profiler`'s serialization format.

### Subprocess injection

Lazyline injects a `sitecustomize.py` file via `PYTHONPATH`
manipulation to bootstrap profiling in child processes. This modifies
the execution environment of all Python subprocesses spawned during a
profiling run. The injected code imports and registers profiling hooks
for the specified scope only.

### Code execution

The `lazyline run ... -- python -c "<code>"` mode uses `exec()` to
execute user-provided code, mirroring CPython's own `-c`
implementation. This is intentional and does not introduce risk beyond
what `python -c` itself provides.

### Input sanitization

- JSON import strips ANSI escape sequences to prevent terminal escape
  injection from crafted profile files.
- NaN and Infinity values are rejected during JSON deserialization.
- Source file reads in the reporting pipeline are restricted to `.py`
  and `.pyw` files.

### No network access

Lazyline makes no network requests. All operations are local.
